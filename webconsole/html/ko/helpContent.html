Configuration Manager 도움말

Sun Java[TM] Desktop System Configuration Manager 도움말 페이지입니다. Java Desktop System Configuration Manager의 특정 부분에 대한 도움말을 보려면 왼쪽에 있는 링크 중 하나를 클릭하십시오.

도움말 페이지의 첫 항목인 배경 및 개념에서는 Configuration Manager 제품에 대한 일반적인 배경 정보와 개념에 대해 설명합니다. 레이아웃 및 동작에서는 Configuration Manager의 일반적인 레이아웃과 동작에 대해 살펴볼 수 있습니다.

일반적인 정보 페이지를 살펴본 후 마스트헤드, 탐색 창 및 내용 창에 대한 보다 구체적인 도움말을 볼 수 있습니다. 로그인과 보고 기능이 있는 페이지도 있습니다.

배경 및 개념

Configuration Manager에서는 구성 설정을 중앙 집중식으로 관리하는 동시에 조직 계층 구조의 각 수준에서 구성 설정을 확인, 정의 및 적용할 수 있습니다. 본 설명서에서는 의미상 긴밀히 연관된 구성 설정을 "구성 정책"이라고 부릅니다.

Configuration Manager는 트리라고도 하는 세 가지 계층 구조를 관리합니다. Configuration Manager의 사용자 인터페이스를 이해하려면 다음 두 트리를 구분하는 것이 중요합니다.

데이터 구조

구성 정책에 의해 만들어진 트리를 구성 정책 트리라고 합니다(그림 1 - "트리"의 파란색 부분 참조). Mozilla와 같이 구성이 복잡한 프로그램을 사용한 적이 있으면 구성 설정의 계층 구조가 낯설지 않을 것입니다. 일반 구성 정책을 "/Mozilla/Navigator"의 "/Mozilla/Navigator/HomeUrl" 구성 설정에 비유할 수 있습니다.

구성 정책 트리는 두 번째 트리 유형인 조직 트리로 통합됩니다(그림 1 - "트리"의 회색 부분 참조). 조직 트리는 조직 단위 간의 관계를 나타냅니다. 이렇게 해서 "구성 정책 트리가 포함된 조직 트리"가 만들어집니다. 조직 트리는 LDAP 서버를 사용하여 표현됩니다.

세 번째 트리 유형은 도메인 트리입니다. 도메인 트리는 하드웨어 장치 간의 관계를 나타냅니다. 구성 정책 트리는 도메인 트리에 통합되므로 구성 정책 트리가 포함된 도메인 트리가 만들어집니다. 도메인 트리는 LDAP 서버를 사용하여 표현됩니다.

조직 트리와 도메인 트리 사용을 위한 기본 개념은 동일합니다. 두 트리의 주요 차이점은 조직 트리는 사용자를 유지 관리하는 반면 도메인 트리는 호스트를 유지 관리한다는 것입니다. 사용자와 호스트가 두 개의 트리에서 별도로 유지 관리되므로 Configuration Manager는 사용자 기반 구성과 호스트 기반 구성을 모두 제공할 수 있습니다. 두 트리의 유사점 때문에 이 절에서는 조직 트리를 중심으로 설명하며, 조직 트리와 도메인 트리 간에 차이점이 있는 경우에만 도메인 트리에 대해 언급합니다.

대기업에서 조직 데이터를 계층 구조에 따라 매핑하는 경우 주로 LDAP 서버를 사용합니다. LDAP에서 조직 트리 내의 각 위치를 엔티티라고 합니다.

 그림 1 – 트리

보안 및 데이터 관리

이러한 "트리의 트리" 구조는 계층적 보안 및 데이터 관리를 용이하게 하며 초기에 전체 구성 정책 트리를 동시에 조직 트리의 루트에 삽입함으로써 만들어집니다. 초기 구성 정책 트리의 구성 설정에 대한 엔티티별 변경 사항은 해당 엔티티에 저장됩니다. 쉽게 알아볼 수 있도록 그림 1에는 "marketing"과 "jclarke"라는 조직 엔티티와 관련된 구성 정책 트리만 표시되어 있습니다.

구성 설정 생성

특정 엔티티에 대한 구성 설정은 병합에 의해 생성됩니다. 조직 트리나 도메인 트리에서 특정 엔티티와 관련된 모든 구성 설정을 레이어라고 합니다. 병합은 루트 레이어에서 시작되며 원하는 엔티티에 도달할 때까지 트리를 따라 진행됩니다. 특정 엔티티와 관련된 레이어에서 변경된 구성 설정은 트리 계층 구조에서 하위 엔티티와 관련된 레이어에 상속됩니다(그림 2 - "레이어 병합" 참조). 특정 레이어에서 자체 설정을 정의하면 상속된 값을 덮어쓸 수 있습니다.

 그림 2 – 레이어 병합

구성 설정과 마찬가지로 보안 특징도 상속됩니다."o=marketing"에 연결된 레이어에서 특정 구성 설정에 대해 "보호됨" 보안 속성을 지정하면 트리에서 모든 후속 레이어의 이 구성 설정이 읽기 전용으로 설정됩니다(그림 3 - "보호" 참조).

 그림 3 – 보호

조직 트리 또는 도메인 트리와 구성 정책 트리 연결

구성 정책 트리는 다음 두 가지 방법으로 조직 트리의 엔티티에 연결됩니다.

  • 독립형 구성 정책 트리로 특정 엔티티에 저장되는 경우
  • 정책 그룹에서 특정 엔티티로 연결되는 경우

정책 그룹

정책 그룹은 구성 정책의 컨테이너로, 각각 고유한 이름을 갖습니다. 구성 정책이 해당 구성 설정을 구성 정책 트리에 저장하기 때문에 정책 그룹은 구성 정책 트리의 컨테이너이기도 합니다(그림 1 – "트리" 참조). 정책 그룹은 정책 리포지토리에 저장되며, 정책 그룹에 대한 링크를 만들어 모든 유형의 엔티티가 사용할 수 있으므로 사용된 백엔드와 관계 없이 유연성이 증가합니다. 정책 그룹의 또 다른 이점은 가져오기와 내보내기가 가능하기 때문에 유지 관리가 용이하다는 것입니다.

관리자는 정책 리포지토리에 저장된 "Experts" 및 "Novice" 정책 그룹과 같이 의미상 구분되는 범위에 대해 구성 정책 집합을 만든 후 전체 조직 트리에서 사용할 수 있습니다.

맨 위로
레이아웃 및 동작

Configuration Manager의 레이아웃은 마스트헤드, 탐색 창 및 내용 창의 세 부분으로 이루어져 있습니다. 마스트헤드는 페이지 맨 위에 있습니다. 왼쪽 아래에는 탐색 창, 오른쪽 아래에는 내용 창이 위치해 있습니다. 이러한 레이아웃 유형을 왼쪽-오른쪽 창 집합이라고도 합니다. 추가 대화 상자와 도움말 내용을 표시하기 위해 별도의 브라우저 창이 열립니다.

맨 위로
로그인

Configuration Manager를 사용하려면 Sun Web Console에 로그인해야 합니다. Sun Web Console을 통해 액세스하는 관리 응용 프로그램은 모두 동일한 사용자 인터페이스를 제공합니다. 콘솔의 가장 큰 이점은 관리자가 한 번만 로그인하면 콘솔 내에서 모든 응용 프로그램을 사용할 수 있다는 것입니다. 이런 목적을 위해 Sun Web Console은 표준 로그인 페이지를 제공합니다.

표준 로그인 페이지에서 사용자 이름과 암호 텍스트 필드 위에는 현재 로그인하고 있는 서버가 표시됩니다.

Web Console에 성공적으로 로그인하면 Web Console 응용 프로그램 시작 페이지가 나타납니다. 여기서 "Configuration Manager" 응용 프로그램을 선택하면 Configuration Manager가 시작됩니다.

맨 위로
마스트헤드

마스트헤드에는 전체 제품 이름(Sun Java[TM] Desktop System Configuration Manager)과 많은 일반적인 링크가 표시되어 있습니다.

  • 콘솔 링크를 클릭하면 Sun Web Console 응용 프로그램 시작 페이지로 돌아갑니다.
  • 버전 링크를 클릭하면 버전 창이 열립니다.
  • 로그 아웃 링크를 클릭하면 Sun Web Console과 Configuration Manager에서 로그아웃되고 로그인 페이지로 돌아갑니다.
  • 도움말 링크를 클릭하면 이러한 도움말 페이지가 열립니다.
맨 위로
탐색 창

탐색 창에서는 조직, 사용자 및 역할이 포함된 조직 트리와 도메인 및 호스트가 포함된 도메인 트리를 탐색할 수 있습니다. 또한 해당 사용자 및 호스트 정책 리포지토리에 있는 정책 그룹을 관리할 수도 있습니다.

탐색 창에는 두 개의 최상위 탭인 사용자 탭과 호스트 탭이 있습니다.

사용자 탭

두 개의 하위 탭인 조직 트리 탭과 정책 리포지토리 탭이 있습니다. 조직 트리 탭을 사용하여 조직 트리를 탐색할 수 있습니다.

상하 경로

조직 트리 탭의 맨 위에는 현재 엔티티 경로를 시각적으로 보여주는 컨트롤인 상하 경로가 있습니다. 모든 경로 구간은 특정 엔티티를 나타내는 링크입니다. 단, 일반 텍스트로 표시되고 현재 엔티티를 나타내는 마지막 링크는 제외됩니다. 상하 경로를 사용하여 탐색하기 위해 특정 링크를 클릭하면 다음 작업이 수행됩니다.

  • 상하 경로가 클릭한 엔티티를 가리키고 탐색 테이블에 해당 엔티티의 하위 엔티티가 표시되도록 탐색 창이 새로 고쳐집니다.
  • 해당 엔티티와 관련된 구성 정책 데이터가 내용 창에 표시됩니다.
탐색 창과 내용 창은 전체 응용 프로그램에서 동기화되어 동일한 데이터 보기를 제공합니다.

탐색 테이블

탐색 테이블은 상하 경로 아래에 있으며 현재 엔티티의 하위 엔티티가 나열됩니다. "이름" 열에는 현재 엔티티의 모든 하위 엔티티 이름이 포함됩니다. "유형" 열에는 엔티티 유형이 표시됩니다. 모든 행의 "작업" 열에는 보기 링크가 있습니다.

특정 엔티티로 이동하기 위해 나열된 이름 중 하나를 클릭하면 다음 작업이 수행됩니다.

  • 선택한 하위 엔티티가 현재 엔티티가 됩니다.
  • 상하 경로가 새로운 현재 엔티티를 가리키고 탐색 테이블에 해당 엔티티의 하위 엔티티가 표시되도록 탐색 창이 새로 고쳐집니다.
  • 해당 엔티티와 관련된 구성 정책이 내용 창에 표시됩니다.

엔티티 유형은 "조직", "사용자" 또는 "역할" 중 하나입니다.

나열된 특정 엔티티의 세부 정보를 보기 위해 보기 링크를 클릭하면 다음 작업이 수행됩니다.

  • 선택한 행의 배경이 파란색으로 바뀝니다.
  • 선택한 엔티티와 관련된 데이터가 내용 창에 표시됩니다.
배경이 파란색인 행은 현재 내용 창에 표시된 데이터의 엔티티를 나타냅니다.

탐색 테이블의 작업 표시줄에는 필터 드롭다운 메뉴와 고급 필터 아이콘에 있습니다. 테이블 항목 수가 10개를 초과하면 테이블 항목 보기를 변경할 수 있도록 페이지 단위 표시/스크롤 아이콘이 표시됩니다.

필터 드롭다운 메뉴에서 탐색 테이블에 표시할 항목 유형을 선택할 수 있습니다. 다음 항목 중에서 선택할 수 있습니다.

  • "모든 항목"은 모든 유형의 엔티티를 탐색 테이블에 표시합니다.
  • "조직"은 "조직" 유형의 엔티티만 탐색 테이블에 표시합니다.
  • "사용자"는 "사용자" 유형의 엔티티만 탐색 테이블에 표시합니다.
  • "역할"은 "역할" 유형의 엔티티만 탐색 테이블에 표시합니다.

고급 필터

탐색 테이블에서 고급 필터 아이콘을 클릭하면 고급 필터 대화 상자가 열립니다.

유형 섹션에서 필터링할 엔티티 유형을 표시합니다. 보다 구체적인 필터를 만들려면 이름 텍스트 필드에 필터를 적용할 이름을 입력한 다음 대화 상자 아래쪽에 있는 필터 버튼을 클릭하여 필터를 실행합니다.

검색

검색 창을 열려면 탐색 테이블 위에 있는 검색 버튼을 클릭합니다. 검색 창은 마스트헤드, 매개 변수 영역(왼쪽) 및 결과 영역(오른쪽)으로 이루어져 있으며 조직 트리 전체에서 특정 엔티티 유형과 엔티티 이름을 반복해서 검색하는 데 사용할 수도 있습니다. 검색 창이 백그라운드에 열려 있을 때 주 창의 탐색 창에서 검색 버튼을 클릭하면 검색 창이 맨 위에 표시됩니다. 검색 창 매개 변수 영역의 상하 경로에 표시되는 검색 창의 현재 엔티티가 새로 고쳐집니다. 다른 모든 매개 변수와 결과 영역의 내용은 변경되지 않습니다.

매개 변수 영역의 맨 위에는 현재 엔티티 경로를 시각적으로 보여주는 상하 경로가 표시됩니다. 매개 변수 영역의 사용 방법에 대해서는 상하 경로 절에서 설명합니다.

특정 엔티티 유형을 검색하려면 상하 경로 아래에 있는 드롭다운 목록에서 해당 유형을 선택합니다. "모두 검색", "조직 검색", "사용자 검색", "역할 검색", "도메인 검색", "호스트 검색" 등의 6개 유형 중에서 선택할 수 있습니다.

검색 범위를 좁히려면 드롭다운 목록 옆에 있는 텍스트 상자를 사용합니다. 이 필드에 입력한 검색 문자열과 일치하는 엔티티 이름을 검색할 수 있습니다. 시작 위치 라디오 버튼 중 하나를 선택하여 트리 검색을 시작하는 위치를 지정할 수도 있습니다. 현재 위치부터 검색하거나 트리 맨 위에서 검색을 시작할 수 있습니다. 기본값은 현재 위치에서 검색하는 것입니다. 각 위치는 해당 이름으로 표현됩니다. 상하 경로에서 특정 경로 구간을 클릭하면 현재 검색 위치가 선택한 엔티티로 바뀌지만 네비게이터의 표시 영역은 변경되지 않습니다.

"페이지 당 표시되는 결과 수" 드롭다운 목록에서 표시할 결과 수를 선택합니다.

검색 창에 있는 모든 검색 필드는 검색 문자열에 와일드카드로 사용된 별표("*")를 허용합니다. 별표는 임의 수의 임의 문자를 나타냅니다. 검색 조건을 여러 개 지정하면 모든 검색 조건에 일치하는 엔티티만 결과 테이블에 추가됩니다. 검색은 대/소문자를 구분하지 않습니다.

검색을 시작하려면 검색 버튼을 클릭합니다. 그러면 검색 결과가 포함된 테이블이 결과 영역에 표시됩니다.

페이지에서 변경한 값을 원래대로 복원하려는 경우 원래대로 버튼을 클릭하면 매개 변수 영역에 기본값이 표시됩니다. 텍스트 상자에 입력한 값은 모두 지워지고 드롭다운 목록과 라디오 버튼 그룹에서는 첫 번째 값이 선택됩니다.

검색 창을 처음 실행하면 결과 영역에 빈 페이지가 나타나고 검색 후에 결과 테이블이 표시됩니다. 결과 테이블에는 "이름", "유형" 및 "경로" 열이 있습니다.

결과를 보기 위해 "이름" 열 아래에서 해당 이름을 클릭하면 다음 작업이 수행됩니다.

  • 주 창이 맨 위에 표시됩니다.
  • "이름" 링크가 나타내는 엔티티가 탐색 테이블에서 파란색 배경으로 표시되도록 주 창의 탐색 창이 새로 고쳐집니다.
  • 해당 엔티티와 관련된 구성 정책이 주 창의 내용 창에 표시됩니다.

"유형" 열에는 엔티티 유형이 표시됩니다. 엔티티 유형은 "조직", "사용자" 또는 "역할" 중 하나입니다.

"경로" 열에는 엔티티 경로가 포함됩니다. 이 경로는 검색 시작 위치에 상대적인 경로를 표시합니다.

"사용자" 유형을 검색하는 경우 "사용자 ID", "이름", "성" 등의 사용자 정보를 입력하기 위한 여러 개의 텍스트 상자가 매개 변수 영역에 표시됩니다. 결과 영역에도 "이름" 열 오른쪽에 사용자 ID가 포함된 "사용자 ID" 열이 추가됩니다. 매개 변수 영역의 "사용자 ID" 텍스트 상자에 최소 한 문자 이상을 입력하면 "사용자 ID" 열이 기본 정렬 기준이 됩니다. 그렇지 않으면 "이름" 열이 기본 정렬 기준이 됩니다.

호스트 탭에는 두 개의 하위 탭인 도메인 트리 탭과 정책 리포지토리 탭이 있습니다.

도메인 트리 탭

도메인 트리에는 사용하고 있는 하드웨어의 구성 설정이 표시됩니다.

도메인 트리의 탐색 방법은 조직 트리와 동일합니다. 자세한 내용은 사용자 탭 절을 참조하십시오.

작업 표시줄

탐색 테이블의 필터 드롭다운 메뉴에는 다음 4개 항목이 포함되어 있습니다.

  • "모든 항목" - 모든 엔티티 유형을 표시합니다.
  • "도메인" - 도메인 유형의 엔티티를 표시합니다.
  • "호스트" - 호스트 유형의 엔티티를 표시합니다.

고급 필터

정책 리포지토리 탭은 사용자 탭과 호스트 탭에 모두 있습니다.

정책 리포지토리는 사용자 정책 그룹이나 호스트 정책 그룹의 컨테이너입니다. 정책 그룹은 우선 순위에 따라 정의된 순서대로 나열됩니다.

정책 그룹 테이블

정책 그룹 테이블은 페이지 맨 위에 있으며 정책 그룹이 나열됩니다. 이 테이블에는 선택 열, "이름" 및 "우선 순위" 열이 있습니다.

선택 열은 정책 그룹 작업 드롭다운 메뉴에 나열된 작업을 적용할 행을 표시하는 데 사용됩니다.

특정 정책 그룹으로 이동하기 위해 해당 이름을 클릭하면 다음 작업이 수행됩니다.

  • 선택한 행의 배경이 파란색으로 바뀝니다.
  • 선택한 정책과 관련된 데이터가 내용 창에 표시됩니다.

"우선 순위" 열에는 정책 그룹의 우선 순위가 포함됩니다. 관리자가 특정 엔티티에 두 개 이상의 정책 그룹을 연결한 경우 우선 순위에 따라 정책 그룹의 병합 순서가 정의됩니다.

배경이 파란색인 행은 현재 내용 창에 표시된 데이터의 정책 그룹(표시된 정책 그룹)을 나타냅니다.

정책 그룹 우선 순위

정책 그룹 우선 순위의 개념은 관리자가 레이어 병합 순서를 정의할 수 있도록 하는 것입니다. 특정 엔티티에 정책 그룹이 두 개 이상 지정되어 있으면 병합 과정에서 정책 그룹 우선 순위가 사용됩니다. 이 경우 엔티티 계층 구조만으로는 정책 그룹 병합 순서를 결정하기에 충분하지 않으므로 정책 그룹에 우선 순위를 지정하여 문제를 해결합니다.

정책 그룹 우선 순위 대화 상자를 열려면 정책 그룹 작업 드롭다운 메뉴에서 우선 순위 편집을 선택합니다. 정책 그룹의 우선 순위를 높이거나 낮추려면 다음 작업을 수행합니다.

  1. 목록에서 정책 그룹을 선택합니다.
  2. 위로 이동 또는 아래로 이동 버튼을 클릭하여 우선 순위를 높이거나 낮춥니다.

내용 창

내용 창에는 탐색 창에서 선택한 엔티티나 정책 그룹과 관련된 데이터가 표시됩니다. 데이터는 탭으로 묶여 있습니다. 탐색 창에서 선택한 항목에 따라 내용 창에 표시되는 탭의 수와 유형이 결정됩니다. 기본 활성 탭은 정책 탭입니다. 현재 활성 탭은 탐색 창에서 다른 항목을 선택해도 새 항목에서 해당 탭을 제공하기만 하면 계속 활성 상태를 유지합니다. 새로 선택한 항목에서 해당 탭을 제공하지 않으면 정책 탭이 활성화됩니다. 탭을 다시 활성화하면 해당 탭의 내부 상태(상하 경로, 정렬 순서)도 그대로 불러옵니다.

정책 탭

정책 탭에서는 하위 그룹, 구성 설정 또는 둘 다를 표시하는 구성 정책 트리를 탐색할 수 있습니다.

모든 정책 페이지에는 다음 두 개의 버튼이 있습니다.

  • 보고서: 자세한 내용은 보고 절을 참조하십시오.
  • 설정 지우기: 이 버튼은 현재 표시된 엔티티 하위 그룹의 모든 구성 데이터를 재귀적으로 삭제합니다.

구성 정책 트리에서 각 루트 항목은 "StarSuite 7"과 같은 응용 프로그램을 나타냅니다. 응용 프로그램 아래의 트리는 해당 응용 프로그램에 속하는 구성 정책을 나열합니다. 구성 정책은 구성 정책 트리의 어디에나 위치할 수 있습니다. 구성 정책이 구성 정책 트리의 리프가 아니면 구성 정책 트리에서 현재의 한 위치에 대한 정책 테이블과 구성 정책을 모두 표시해야 합니다.

상하 경로

상하 경로는 페이지 맨 위, 탭 아래에 표시되며 구성 정책 트리에서의 현재 위치를 시각적으로 보여줍니다. 또한 탐색 창의 상하 경로와 동일한 방식으로 작동합니다(이전 상하 경로 절 참조).

하위 그룹 테이블

하위 그룹 테이블은 상하 경로 아래에 있으며 구성 정책 트리에서 현재 위치의 하위 그룹이 나열됩니다. 이 테이블에는 "이름" 열과 "설명" 열이 있습니다.

"이름" 열에는 구성 정책 트리에서 현재 위치의 모든 하위 그룹 이름이 포함됩니다. 각 이름은 링크로 표시됩니다. 구성 정책 트리를 탐색하기 위해 특정 이름 링크를 클릭하면 다음 작업이 수행됩니다.

  • 상하 경로가 구성 정책 트리에서의 새 위치를 가리키도록 내용 창이 새로 고쳐집니다.
  • 정책 테이블이나 정책이 표시되도록 내용 창이 새로 고쳐집니다.

이러한 탐색 방법을 "드릴다운" 탐색이라고 합니다.

설명 열에는 하위 그룹에 대한 간략한 설명이 포함됩니다.

정책

현재 엔티티와 현재 구성 정책의 조합에 사용할 수 있는 유효한 구성 정책 데이터가 있으면 내용 창의 "정책" 탭에 해당 데이터가 표시됩니다.

데이터는 테이블로 제공되며, 각 테이블에는 선택 아이콘이 포함된 선택 열, "상태" 열, "이름" 열, "값" 열 등 모두 4개의 열이 있습니다. 테이블 작업 표시줄에는 드롭다운 작업 메뉴가 있습니다.

특정 요소를 선택하려면 앞에 있는 확인란을 선택합니다. 이제 작업 표시줄에 있는 드롭다운 메뉴의 작업을 선택한 요소에 대해 수행할 수 있습니다. 다음 표에는 각 작업에 대한 자세한 설명이 나와 있습니다.

조직(또는 도메인) 트리의 상위 수준에서 요소 보호가 설정되었음을 나타내며 링크를 제공합니다. 이 요소 또는 항목의 보호는 조직 내의 레이어, 즉 엔티티 수준의 병합 결과입니다.

아이콘을 클릭하면 보호가 설정된 위치로 이동합니다.

"값" 열의 값을 변경하여 데이터 값을 변경할 수 있습니다. 상태 변경 사항과 마찬가지로 변경된 값도 저장해야 합니다. 저장 버튼을 클릭하면 값이 저장됩니다.

집합

집합은 동적 등록 정보 컬렉션을 의미합니다. 요소를 집합에 추가하고 제거할 수 있습니다.

요소를 집합에 추가하려면 다음 작업을 수행합니다.

  1. 새로 만들기 버튼을 클릭합니다.
  2. 새 요소 이름을 입력할 수 있는 대화 상자가 열립니다. 요소가 추가되고 주 창이 새로 고쳐집니다.
  3. 주 창에서 새 요소를 편집할 수 있습니다.

요소를 집합에서 삭제하려면 해당 요소를 선택하고 삭제 버튼을 클릭합니다.

집합은 링크로 표현되며 다른 집합에 포함될 수도 있습니다. 링크를 클릭하면 집합으로 이동하며 여기서 집합을 편집할 수 있습니다.

정책 그룹 탭

정책 그룹 페이지에서는 정책 그룹을 선택한 엔티티에 추가하고 제거할 수 있습니다.

왼쪽의 사용 가능 목록에 있는 정책 그룹을 추가하려면

  1. 목록에서 정책 그룹을 선택합니다.
  2. 추가 버튼을 클릭하여 선택한 정책 그룹을 오른쪽의 선택 목록에 추가합니다.
  3. 저장을 클릭하여 새 지정을 저장합니다.
지정 대상 탭

탐색 창의 정책 리포지토리에서 정책 그룹을 선택하면 내용 창에 지정 대상 탭 페이지가 포함됩니다. 지정 대상 페이지에는 선택한 정책 그룹이 지정된 엔티티가 모두 나열됩니다.

지정 대상 탭 페이지에서 수행할 수 있는 작업은 다음과 같습니다.

  • 제거는 선택한 엔티티와 탐색 창에서 선택한 정책 그룹 간의 연결을 끊습니다.
  • "이름" 열에서 특정 엔티티를 클릭하면 탐색 창에서 해당 엔티티 배경이 파란색으로 표시되도록 탐색 창이 새로 고쳐집니다.

역할 탭

탐색 창에서 "사용자" 엔티티 유형을 선택하고 해당 사용자가 최소 한 개 이상 역할의 구성원이면 내용 창에 역할 탭 페이지가 포함됩니다. 역할 페이지에는 선택한 사용자가 구성원으로 속해 있는 역할이 모두 나열됩니다.

이 페이지에는 "이름" 열과 "경로" 열이 있습니다. "이름" 열에는 역할 이름이 포함되고 "경로" 열에는 역할에 대한 절대 경로가 포함됩니다.

보고

보고서는 데이터가 포함된 모든 구성 설정의 읽기 전용 보기입니다. 보고서 버튼을 클릭하면 보고서가 트리거되고 보고서 구성 대화 상자가 나타납니다.

"보고서 구성" 대화 상자에서는 다음 설정을 지정할 수 있습니다.

  • 보고 시 고려할 트리(조직 및/또는 도메인 트리)(보고서 용도)
  • 보고서 작성 시 표시할 열(상태 경로 열과 설명 열은 비활성 가능)

사용자 정의가 끝난 후 보고서를 클릭하면 선택한 데이터만 읽기 전용으로 표시됩니다.
